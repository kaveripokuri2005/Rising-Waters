# Entity Relationship (ER) Diagram

## Overview

The Entity Relationship (ER) Diagram illustrates the database structure of the Flood Prediction System. It represents the main entities, their attributes, and the relationships between them. This design helps organize data efficiently, maintain data integrity, and support the machine learning-based flood prediction process.

---

## Purpose

The ER diagram is designed to:

- Organize weather and prediction data efficiently.
- Define relationships between users, weather records, machine learning models, and prediction results.
- Ensure consistency and integrity of stored information.
- Support future scalability and deployment of the application.

---

## Entities

### 1. USER

Stores information about authorized users of the system.

**Attributes:**
- User_ID (Primary Key)
- Name
- Email
- Password
- Role (Administrator / Meteorologist / Disaster Officer)

**Purpose:**
Users can enter weather information and view flood prediction results.

---

### 2. WEATHER_DATA

Stores the meteorological parameters used for flood prediction.

**Attributes:**
- Weather_ID (Primary Key)
- User_ID (Foreign Key)
- Annual_Rainfall
- Seasonal_Rainfall
- Cloud_Visibility
- Temperature
- Humidity
- Input_Date

**Purpose:**
This data serves as input for the machine learning model.

---

### 3. PREDICTION

Stores the prediction generated by the machine learning model.

**Attributes:**
- Prediction_ID (Primary Key)
- Weather_ID (Foreign Key)
- Model_ID (Foreign Key)
- Flood_Risk (Yes/No)
- Probability
- Prediction_Date

**Purpose:**
Stores the predicted flood risk and confidence score.

---

### 4. ML_MODEL

Stores information about the machine learning models used in the application.

**Attributes:**
- Model_ID (Primary Key)
- Model_Name
- Algorithm
- Accuracy
- Version

**Example Algorithms**
- Decision Tree
- Random Forest
- K-Nearest Neighbours (KNN)
- XGBoost

**Purpose:**
Maintains details of the trained models used to generate predictions.

---

## Relationships

| Relationship | Description |
|--------------|-------------|
| USER → WEATHER_DATA | One user can enter multiple weather records. |
| WEATHER_DATA → PREDICTION | Each weather record generates one prediction. |
| ML_MODEL → PREDICTION | One machine learning model can generate multiple predictions. |

---

## Workflow

1. A user logs into the system.
2. Weather parameters are entered.
3. The trained machine learning model processes the input.
4. The system predicts whether a flood is likely to occur.
5. The prediction result is stored and displayed through the Flask web application.

---

## Benefits of the ER Diagram

- Improves database organization.
- Reduces data redundancy.
- Maintains relationships between entities.
- Supports efficient querying and data management.
- Provides a clear blueprint for database implementation.

---

## Conclusion

The ER Diagram provides a structured representation of the Flood Prediction System's database. It establishes the relationships between users, weather data, prediction results, and machine learning models, enabling efficient storage, retrieval, and management of information required for accurate flood prediction.
